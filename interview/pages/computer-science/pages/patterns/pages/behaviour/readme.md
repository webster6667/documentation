# Поведенческие паттерны
> Отвечают за объединение или расширения структуры моделей

<details>
<summary> 🔥 <code>Shortcut</code></summary>

___

🔹 Цепочка обязанностей  
&emsp;&emsp; 👆 Позволяет вызывать методы инстанса цепочкой из результата работы других методов




🔹 Команда  
&emsp;&emsp; 👆 Оболочка позволяющая управлять функционалом через другой объект, исключив прямые изменения инстанса, только при помощи экшенов  
&emsp;&emsp;&emsp;&emsp; 📗 Хорошим примером command патерна являеться `Redux`, в нем нельзя просто на прямик изменить `store`, для этого нужно выполнить цепочку команд





🔹 Итератор  
&emsp;&emsp; 👆 Дает возможность последовательно обходить элементы составным объектов, не раскрывая их внутреннее представление
&emsp;&emsp;&emsp;&emsp; 📗 Как это делает `for(..of..)`


🔹 Медиатор  
&emsp;&emsp; 👆 Жесткая связи между двумя разными инстансами класса

🔹 Наблюдатель  
&emsp;&emsp; 👆 Патерн позволяющий следить одному объекту за изменением другово



🔹 Состояние  
&emsp;&emsp; 👆 Разное поведение одних и тех же хендлеров, в зависимости от состояние инстанса(которое можно менять)



🔹 Стратегия  
&emsp;&emsp; 👆 Оболочка взаимодействующая с различными стратегиями, используя один и то же интерфейс
```javascript
commute.travel(new Taxi())
commute.travel(new Bus())
commute.travel(new Car())
```


🔹 Шаблонный метод  
&emsp;&emsp; 👆 Патерн определяющее шаблонное выполнение алгоритмов(порядов выполнения), имеющих одинаковые название, но разную реализацию

```javascript
class Builder {
    build() {
        this.addEngine();
        this.installChassis();
        this.addElectronic();
        this.collectAccessories();
    }
};

// 👉🏼 От билдера наследуються классы по своему реализующие каждый из методов внутри билда
```
